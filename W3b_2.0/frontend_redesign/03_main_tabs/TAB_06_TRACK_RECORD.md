# Main Tab 6: Track Record (Verified Alpha)

## The Objective
This is the proof-of-work tab. While the Vault shows current capital, the Track Record shows historic immunity to market variance. It is designed to silence doubters with undeniable quantitative transparency. 

## Language & Nomenclature Shift
- **❌ BANNED RETAIL TERMS:** Win/Loss Record, Streak, ROI (used loosely).
- **✅ REQUIRED INSTITUTIONAL TERMS:** Verified Alpha, Model Calibration, Max Drawdown, Sharpe Ratio, Information Ratio, Historical Efficacy.

## Visual Implementation Plan
1. **The Immutability Ledger:**
   - A scrolling, indisputable ledger of resolved event contracts. It must look like raw, terminal-grade system output rather than a fluffy "history" page. Each line contains the Exact Entry Confidence, the Fired Resolution, and the Actual Resolution.
2. **Calibration Scatter Plot:**
   - Comparing the W3B assigned probability against Kalshi's market consensus. Visually proving that when the model says "72% probability," that event statistically happens exactly 72% of the time.
3. **KPI Glass Panels:**
   - Institutional metrics elegantly displayed inside `<InstitutionalGlass />`: Win Rate vs. Expected Value, Cumulative Alpha, and Total Number of Events Traded (N-size).
