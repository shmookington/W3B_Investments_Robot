# Main Tab 8: Docs (Quantitative Methodologies)

## The Objective
Institutional transparency. This is not a "How to Bet" guide. This is an encyclopedic, Stripe-style technical documentation page covering exactly how the W3B Engine prices risk and ingests information theory.

## Language & Nomenclature Shift
- **❌ BANNED RETAIL TERMS:** How to Use, Guide, FAQ.
- **✅ REQUIRED INSTITUTIONAL TERMS:** API Referenace, Information Theory, Valuation Methodologies, Signal Orchestration, Data Pipeline Schemas.

## Visual Implementation Plan
1. **The Classic Split-Pane:**
   - A beautiful 3-column layout: Navigation on the left, Prose/Explanations in the middle, and simulated/mock Code Snippets (Python/JSON) on the right.
2. **Frosted Code Blocks:**
   - Code snippets are housed in deep Navy frosted panels with perfect Fira Code formatting.
3. **Methodology Modules:**
   - Instead of hiding the math, we flaunt it. Explanations on how Kelly Criterion sizing works, how Kelly parameters are dynamically adjusted for volatility, and how multi-source scraping mitigates single-API failures.
